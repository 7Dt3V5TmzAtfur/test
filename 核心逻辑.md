

## 一句话核心

> **React = "数据变了，页面自动跟着变" 的工具。**

三件套：

| 概念 | 是什么 | 比喻 |
|------|--------|------|
| **state** | 数据本身 | 真相之源 |
| **render** | 数据 → JSX 的映射函数 | 厨师照着菜谱做菜 |
| **re-render** | state 变了 → 自动再调用一次 render | 菜谱改了，厨师重做 |

## 用你打开的文件走一遍

打开 [TodoApp.jsx](file:///Users/a/Downloads/a4/src/components/TodoApp.jsx#L31-L103)，整个组件就三块：

**第 1 块：state 在哪里（L33–L41）**

```jsx
const [todos, setTodos] = useState([...])     // 待办数据
const [filter, setFilter] = useState('all')   // 筛选条件
```

数据**只活在 TodoApp 这一处**。所有子组件都不存数据。

**第 2 块：怎么改 state（L44–L63）**

不是直接 `todos.push(...)`，而是**调 setTodos 给一个新数组**：
- 添加：`[...todos, 新项]`（展开 + 新增）
- 切换：`todos.map(...)`（找到 id 匹配的那项改 done）
- 删除：`todos.filter(...)`（留下 id 不匹配的）

> 关键：永远创建**新数组**，别改原数组。React 比对引用，引用没变它就认为"没改"。

**第 3 块：怎么画 UI（L72–L102）**

`return (...)` 里的 JSX，就是**当 state 是 X 时，页面长这样**的声明。
state 一变 → React 自动再跑一次这个 `return` → 浏览器更新。

## 你正盯着的第 77 行

```jsx
<TodoInput onAdd={addTodo} />
```

这一行是**整个 React 哲学的浓缩**：

```
┌──────────────────────────────────────────┐
│  TodoApp（父，持有 state）                │
│                                          │
│  addTodo 这函数在 TodoApp 里，            │
│  它会改 todos 这个 state。                │
│                                          │
│  把它作为 onAdd prop 传给 TodoInput ↓    │
└────────────┬─────────────────────────────┘
             │ props 往下流
             ↓
┌──────────────────────────────────────────┐
│  TodoInput（子，只负责 UI）               │
│                                          │
│  拿到 onAdd，不知道它具体怎么改 state。  │
│  用户点「添加」按钮时，调用 onAdd(text)。 │
└──────────────────────────────────────────┘
```

**为什么这么绕？** 因为 state 在父组件，事件却发生在子组件（用户点子组件里的按钮）。子组件不能直接改父的 state，所以父把"改 state 的方法"作为函数 prop 传下来，子组件在合适的时机回调它。

这就是 **React 单向数据流**：

```
        state（数据）          向下
        ┌──────────┐          props           ┌──────────┐
        │  TodoApp │  ──────────────────────▶ │ TodoInput│
        │          │                          │          │
        │  setX()  │  ◀────────────────────  │ onAdd()  │
        └──────────┘     向上 回调              └──────────┘
                              (onAdd)
```

## 把核心逻辑压缩成口诀

> **数据往上聚（state 在共同父组件），函数往下传（回调 prop），UI 跟着数据自动重画（re-render）。**

## 几个常见误解

| 误解 | 真相 |
|------|------|
| "改 state 就像改普通变量" | 必须 `setX(新值)`，React 监听的是 setter，不是变量赋值 |
| "props 能随便改" | 不能。props 是只读的，想改就通过回调让父组件改 |
| "父改了 state 子不知道" | 子**自动**知道，因为 props 每次 render 都重新传 |
| "re-render 性能差" | React 用虚拟 DOM 做 diff，只更新真正变了的 DOM 节点 |

## 验证你懂没懂
